# Internationalization (i18n)

If you want your React app to reach people beyond your own language, you need internationalization (i18n). It sounds intimidating, but the basics are straightforward. I am learning this as I go, and here is what I have figured out so far.

## react-i18next

The most popular i18n library for React is `react-i18next`. It builds on `i18next` and gives you hooks and components for translating your UI.

### Setup

```bash
npm install i18next react-i18next
```

```ts
// i18n.ts
import i18n from "i18next";
import { initReactI18next } from "react-i18next";

import en from "./locales/en.json";
import es from "./locales/es.json";

i18n.use(initReactI18next).init({
  resources: {
    en: { translation: en },
    es: { translation: es },
  },
  lng: "en",
  fallbackLng: "en",
  interpolation: {
    escapeValue: false,
  },
});

export default i18n;
```

### Translation Files

```json
// locales/en.json
{
  "greeting": "Hello",
  "farewell": "Goodbye",
  "itemCount": "You have {{count}} items",
  "nav.home": "Home",
  "nav.about": "About"
}
```

```json
// locales/es.json
{
  "greeting": "Hola",
  "farewell": "Adios",
  "itemCount": "Tienes {{count}} articulos",
  "nav.home": "Inicio",
  "nav.about": "Acerca de"
}
```

### Using Translations

```tsx
import { useTranslation } from "react-i18next";

function App() {
  const { t } = useTranslation();

  return (
    <div>
      <h1>{t("greeting")}</h1>
      <p>{t("itemCount", { count: 5 })}</p>
      <nav>
        <a href="/">{t("nav.home")}</a>
        <a href="/about">{t("nav.about")}</a>
      </nav>
    </div>
  );
}
```

The `t` function looks up the translation key. Interpolation with `{{count}}` lets you insert dynamic values.

## Language Switching

```tsx
function LanguageSwitcher() {
  const { i18n } = useTranslation();

  const changeLanguage = (lng: string) => {
    i18n.changeLanguage(lng);
  };

  return (
    <div>
      <button onClick={() => changeLanguage("en")}>English</button>
      <button onClick={() => changeLanguage("es")}>Espanol</button>
    </div>
  );
}
```

## Pluralization

i18next handles plurals based on the count:

```json
{
  "items_one": "You have {{count}} item",
  "items_other": "You have {{count}} items"
}
```

```tsx
<p>{t("items", { count: 1 })}</p>  // "You have 1 item"
<p>{t("items", { count: 5 })}</p>  // "You have 5 items"
```

## RTL Support

Some languages like Arabic and Hebrew are read right to left (RTL). You need to handle this in your CSS:

```tsx
function App() {
  const { i18n } = useTranslation();
  const isRTL = i18n.language === "ar";

  return (
    <div dir={isRTL ? "rtl" : "ltr"} className={isRTL ? "text-right" : "text-left"}>
      {/* Your app */}
    </div>
  );
}
```

In Tailwind CSS, you can use logical properties:

```tsx
// Instead of margin-left or margin-right, use:
<div className="ms-4">  /* margin-inline-start */
<div className="me-4">  /* margin-inline-end */
<div className="ps-4">  /* padding-inline-start */
<div className="pe-4">  /* padding-inline-end */
```

These automatically flip for RTL languages.

## Lazy Loading Translations

For large apps, load translations on demand:

```ts
i18n.use(initReactI18next).init({
  fallbackLng: "en",
  interpolation: { escapeValue: false },
});

// Load language when needed
async function loadLanguage(lng: string) {
  const resources = await import(`./locales/${lng}.json`);
  i18n.addResourceBundle(lng, "translation", resources.default);
  i18n.changeLanguage(lng);
}
```

Internationalization is one of those things that is much easier to build in from the start than to add later. Even if you only support one language now, wrapping your text in `t()` calls makes future translation painless.
