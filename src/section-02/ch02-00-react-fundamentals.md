# Starting the React Journey

Welcome to the part where things get real. You have your JavaScript foundations in place, and now it is time to learn React itself.

I remember my first day with React. I opened the docs, saw JSX for the first time, and thought "this is not JavaScript." It looked like HTML mixed with JS in a way that made no sense. But within a few days, it clicked. And once it clicked, I could not go back to writing UIs any other way.

React changed how I think about building interfaces. Instead of manually updating the DOM piece by piece, I describe what the UI should look like, and React handles the rest. It is a different mindset, and it takes a little practice to get used to.

## What You Will Learn Here

This section covers the fundamentals. The building blocks you need before anything else:

- What React is and why it exists
- How React actually works under the hood
- Setting up a project from scratch
- JSX and why it looks like HTML but is not
- Components, props, and state
- Handling events
- Rendering conditionally and rendering lists

These are not advanced topics. They are the basics. But the basics are where most bugs come from. I have spent hours debugging things that came down to misunderstanding state or misusing keys in a list. So take your time here.

## How to Approach This

My advice: type out every example yourself. Do not copy and paste. There is something about typing code that makes it stick in your brain differently. Then change the examples. Break them. See what happens. That is how you really learn.

Also, do not worry about making something impressive yet. A button that counts clicks is a perfectly fine project when you are learning. The fancy stuff comes later.
